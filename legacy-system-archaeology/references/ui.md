# UI维度考古

## 概述

UI维度考古专注于梳理系统的用户界面，包括页面结构、UI规范、交互流程等。

## 1. 页面分析 (pages.md)

### 分析目标
- 梳理所有页面和路由
- 理解页面功能和布局
- 识别页面间的导航关系

### 输出模板

```markdown
# 页面清单

## 页面总览

| 角色 | 页面数 | 主要功能 |
|------|--------|----------|
| 角色A | N | 功能描述 |
| 角色B | N | 功能描述 |

---

## 1. 公共页面

### 1.1 登录页 (Login.vue)
- **路由**: `/login`
- **文件**: `views/auth/Login.vue`
- **功能**:
  - 用户名/密码登录
  - 记住密码
  - 忘记密码
- **布局**: 居中卡片式
- **跳转**: 登录成功 → 角色对应的工作台

### 1.2 错误页
| 页面 | 路由 | 触发条件 |
|------|------|----------|
| 403 | /403 | 无权限 |
| 404 | /404 | 页面不存在 |
| 500 | /500 | 服务器错误 |

## 2. 角色A页面

### 2.1 工作台首页
- **路由**: `/role-a/dashboard`
- **文件**: `views/role-a/dashboard/Index.vue`
- **功能**:
  - 统计数据概览
  - 待办事项提醒
  - 快捷操作入口
- **布局**: 侧边栏 + 内容区
- **组件**:
  - StatCard - 统计卡片
  - TodoCard - 待办卡片
  - UrgentTodoCard - 紧急待办卡片

### 2.2 列表页
- **路由**: `/role-a/list`
- **功能**:
  - 数据列表展示
  - 搜索筛选
  - 分页
- **布局**: 筛选区 + 表格区
- **操作**: 新增、编辑、删除、导出

### 2.3 详情页
- **路由**: `/role-a/detail/:id`
- **功能**:
  - 详细信息展示
  - 操作历史
  - 相关数据
- **布局**: 标签页式或卡片式

### 2.4 表单页
- **路由**: `/role-a/form`
- **功能**:
  - 数据录入
  - 表单验证
  - 提交/取消
- **布局**: 分步骤表单或长表单
```

## 考古方法

### 信息来源
- 设计系统文档（如有）
- CSS/SCSS文件
- 组件库文档
- 实际页面截图
- 设计稿（Figma/Sketch等）
- 开发者工具（检查元素）

### 分析技巧
1. **从CSS推断设计**: CSS变量、主题配置
2. **从组件推断规范**: 组件Props、默认值
3. **从截图提取颜色**: 取色器工具
4. **从开发者工具测量**: 尺寸、间距、字号

### 工具辅助
- **浏览器插件**: ColorZilla（取色）、Measure（测量）
- **设计工具**: Figma（设计稿）、Zeplin（标注）
- **截图工具**: 系统截图、Snipaste
- **CSS查看**: Chrome DevTools

### 常见陷阱
- 只看首页，忽略其他页面
- 忽略状态变化（hover、active、disabled）
- 忽略错误状态的UI
- 混淆桌面端和移动端样式
- 忽略响应式变化
