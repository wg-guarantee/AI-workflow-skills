Skill 是什么？

不是提示词，不是插件 -- 是 AI 的可复用知识包

一句话理解

Skill 就是一个 Markdown 文件，放在固定目录下，AI 会自动读取并按里面的指令工作。
你可以把它理解为 "教 AI 一个新技能" -- 持久的、自动触发的、可以带脚本的。

一个 Skill 长什么样

~/.claude/skills/ ← 主目录（推荐）
  my-skill/
    SKILL.md ← 必须有，skill 的全部
    scripts/ ← 可选，可执行的工具脚本
    references/ ← 可选，AI 按需读取的参考文档
    assets/ ← 可选，模板、图片等资源文件
也支持: ~/.codebuddy/skills/ | 项目级 .claude/skills/ | .codebuddy/skills/
---
name: my-skill
description: Use when [触发条件]
---
↑ Frontmatter：AI 的触发器。始终在内存里，只占几十字。
# Skill Title
具体指令...
↑ Body：只有触发后才会读取，不触发不占资源。

3 步创建你的第一个 Skill

1

想一个你想让 AI 自动做的事

比如：总结文章、翻译、写邮件、整理笔记、代码审查...

2

创建文件

告诉 AI "帮我创建一个 skill"，或者手动创建 ~/.claude/skills/my-skill/SKILL.md

3

用真实内容测试

在新对话里贴内容，看 AI 是不是自动按你的 skill 处理了。不满意就改 SKILL.md，再试。

三者的区别

📖

Skill

教 AI 方法（知识）

  • 自动触发，不用手动
  • 可以带脚本和资源
  • 可分享给别人
  • 不触发不占上下文
  • 类比：一本菜谱
🔧

MCP 工具

给 AI 能力（工具）

  • 连接外部服务
  • 读写文件、调 API
  • AI 按需调用
  • 不含工作流知识
  • 类比：一把刀
💬

提示词

临时的一次性指令

  • 手动粘贴才有效
  • 只能是纯文字
  • 换个对话就没了
  • 贴进去就全占上下文
  • 类比：口头吩咐

Skill 的四个超能力

⚡

自动触发

你贴一段英文，不说任何指令，AI 自动按 skill 翻译。不用你记得粘贴提示词。

提示词：你不粘贴就没有
📦

带脚本和资源

Skill 可以打包 Python 脚本、参考文档、模板文件。PDF skill 带了完整处理脚本，不用每次重写代码。

提示词：只能是纯文字
💡

上下文高效

装 100 个 skill，平时只有 description 占内存（每个几十字）。需要时才加载完整内容。

提示词：贴进去就全占
🤝

可分享

你的 skill 复制给别人，别人装上就能用你的工作流。跨对话、跨项目、跨人。

提示词：存在你脑子里

渐进加载：装得多不会变慢

始终加载 name + description（每个 skill 几十字） ~100 tokens
触发时加载 SKILL.md body（完整指令） 按需
需要时加载 references/、scripts/（参考文档和脚本） 按需

实际演示：看看你已经装了多少 Skill

在终端运行以下命令，查看你已安装的所有 skill：

bash ~/.claude/skills/skill-creating/scripts/show-skills.sh
# 紧凑模式：
bash ~/.claude/skills/skill-creating/scripts/show-skills.sh --compact
