# doc2md 详细参考文档

## PDF 处理指南

### 内容提取顺序

1. **读取 PDF 文件**
   - Claude Code Read 工具可直接解析 PDF
   - 支持多页文档，按页顺序提取

2. **识别结构元素**
   - 标题：通过字体大小和样式判断层级
   - 段落：通过换行和缩进识别
   - 表格：通过边框和对齐识别
   - 图片：通过图像块识别

3. **处理特殊内容**
   - 页眉页脚：可选择性保留
   - 目录：可转为 `[TOC]` 或省略
   - 脚注：转为引用块或内联

### PDF 表格处理

**简单表格：**
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据 | 数据 | 数据 |
```

**合并单元格：**
```markdown
| 列1 | 列2 |
|-----|-----|
| 跨两列的内容 ||
```
或使用说明：`*注：原表格包含合并单元格*`

### PDF 图片处理

1. 识别图片在文档中的位置
2. 按出现顺序编号（图1、图2...）
3. 结合上下文理解图片用途
4. 生成描述性文字

## DOCX 处理指南

### 样式映射

| Word 样式 | Markdown |
|-----------|----------|
| 标题 1 | `#` |
| 标题 2 | `##` |
| 标题 3 | `###` |
| 正文 | 普通文本 |
| 列表 | `-` 或 `1.` |
| 引用 | `>` |
| 代码 | `` `code` `` |

### 特殊元素处理

**批注：**
```markdown
> **批注 [作者]:** 批注内容
```

**修订记录：**
```markdown
~~删除的内容~~ **新增的内容**
```

**文本框：**
```markdown
> 文本框内容
```

### DOCX 表格处理

- 保持表格结构
- 复杂表格添加结构说明
- 嵌套表格展平处理

## XLSX 处理指南

### 多 Sheet 处理

```markdown
# Sheet 1: 工作表名称

[表格内容]

---

# Sheet 2: 工作表名称

[表格内容]
```

### 数据转换规则

| Excel 元素 | Markdown 处理 |
|------------|---------------|
| 单元格数据 | 直接转为文本 |
| 公式 | 转为计算结果值 |
| 格式化数字 | 保留格式说明 |
| 图表 | 视觉理解后描述 |
| 条件格式 | 添加说明 |

### 大表格处理

**超过 50 行的表格：**
1. 保留表头
2. 提取前 10 行作为示例
3. 添加统计摘要
4. 说明：`*表格共 N 行，以下为前 10 行示例*`

## PPTX 处理指南

### 幻灯片结构

```markdown
# 幻灯片 1: [标题]

[内容]

---

# 幻灯片 2: [标题]

[内容]
```

### 元素处理

| PPT 元素 | Markdown 处理 |
|----------|---------------|
| 标题 | `##` 级标题 |
| 正文 | 普通文本 |
| 项目符号 | 无序列表 |
| 图片 | 视觉描述 |
| SmartArt | 描述流程/关系 |
| 备注 | `> 备注: 内容` |

### 动画和过渡

忽略动画效果，只保留静态内容。

## 图片视觉理解指南

### 分析流程

1. **识别图片类型**
   - 图表/图形
   - 流程图
   - 截图
   - 照片
   - 图标/Logo
   - 公式/方程

2. **提取关键信息**
   - 文字内容（如有）
   - 数据值（图表类）
   - 结构关系（流程图类）
   - 视觉元素（颜色、形状）

3. **生成描述**

## 校验报告详解

### 检查项说明

| 检查项 | 检查方法 | 通过标准 |
|--------|----------|----------|
| 标题数量 | 统计 H1-H6 数量 | 差异 ≤ 1 |
| 段落数量 | 统计非空行数 | 差异 ≤ 2 |
| 表格数量 | 统计表格块数 | 完全一致 |
| 图片数量 | 统计图片描述数 | 完全一致 |
| 关键术语 | 提取专业词汇检查 | 全部保留 |

### 状态定义

- **✅ 完全匹配**：数量一致，内容完整
- **⚠️ 轻微差异**：数量略有差异，核心信息完整
- **❌ 需要修复**：存在明显信息丢失

### 差异报告格式

```markdown
### 差异说明

1. **段落 #5**
   - 原文：包含脚注引用
   - 转换：脚注已内联到正文
   - 影响：无，信息保留完整

2. **图片 #2**
   - 原文：组织架构图
   - 转换：文字描述
   - 影响：轻微，建议人工复核
```

## 常见问题处理

### 复杂表格

**问题：** 表格结构复杂，难以用 Markdown 表达

**解决方案：**
1. 尝试简化表格结构
2. 添加文字说明补充
3. 必要时标注：`*原表格结构复杂，以下为简化版本*`

### 扫描版 PDF

**问题：** PDF 为扫描图片，无法直接提取文字

**解决方案：**
1. Claude 视觉能力可直接读取扫描内容
2. 逐页识别并转换
3. 注意校验识别准确性

### 大文件处理

**问题：** 文档页数过多，超出上下文限制或并发请求限制

**解决方案：两阶段分离处理**

#### 阶段一：文本提取（可逐页处理）

```
1. Read pages: "1" → 提取文本和表格 → 追加到 MD 框架 → 记录图片位置
2. Read pages: "2" → 提取文本和表格 → 追加到 MD 框架 → 记录图片位置
3. Read pages: "3" → 提取文本和表格 → 追加到 MD 框架 → 记录图片位置
...依此类推
```

此阶段只处理文本和表格，**不触发视觉模型调用**，并发风险低。

#### 阶段二：图片处理（严格串行）

```
1. 按记录的图片清单，逐个处理
2. 处理图片1 → 生成描述 → 填充到 MD
3. 等待完成后，处理图片2 → 生成描述 → 填充到 MD
4. 依此类推，每次只处理一张图片
```

**关键注意事项：**
- ⚠️ **阶段一必须完全完成后，才能开始阶段二**
- ⚠️ **图片处理必须严格串行，禁止同时处理多张图片**
- ⚠️ **每张图片处理完成后，确认再处理下一张**
- 校验时使用阶段一缓存的文档信息，避免重新读取

### 特殊字符

**问题：** 文档包含 Markdown 特殊字符

**解决方案：**
- 使用反斜杠转义：`\*`、`\#`、`\[` 等
- 代码块中的特殊字符无需转义
