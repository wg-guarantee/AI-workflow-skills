# 遗留系统知识考古检查清单

## 使用说明

在考古过程中，使用此清单确保不遗漏重要内容。每完成一项，在 `[ ]` 中标记为 `[x]`。

---

## 第一阶段：准备阶段

### 环境准备
- [ ] 确认系统可访问（开发/测试环境）
- [ ] 获取必要的访问权限（数据库、代码仓库、服务器）
- [ ] 确认联系人（业务人员、开发人员、运维人员）

### 信息收集
- [ ] 收集现有文档（需求文档、设计文档、用户手册）
- [ ] 收集代码仓库地址和访问权限
- [ ] 收集数据库访问信息
- [ ] 收集测试账号信息

---

## 第二阶段：业务维度考古

### 角色分析
- [ ] 识别所有用户角色
- [ ] 理解每个角色的职责
- [ ] 梳理角色的菜单权限
- [ ] 理解角色的数据权限隔离规则
- [ ] 生成 `roles.md`

### 流程分析
- [ ] 识别核心业务流程（至少3个）
- [ ] 梳理每个流程的参与角色
- [ ] 绘制流程图（泳道图）
- [ ] 列出流程中的关键步骤
- [ ] 识别异常处理流程
- [ ] 生成 `business-flow-analysis.md` 和 `workflows.md`

### 状态机分析
- [ ] 识别有状态的业务对象
- [ ] 列出每个对象的所有状态
- [ ] 绘制状态流转图
- [ ] 列出状态转换的触发条件
- [ ] 生成 `status-machines.md`

---

## 第三阶段：技术维度考古

- [ ] 理解项目结构（目录、模块、分层）

- [ ] 梳理Controller层（API入口）

- [ ] 梳理页面组件（按角色/功能）
- [ ] 梳理路由配置
- [ ] 梳理状态管理

### 数据库分析
